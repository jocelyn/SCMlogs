Quick description

  • Sending or not

    • You decide if you want to receive or not the logs emails.

      - If you check "Email sending" then you will receive emails, otherwise you won't receive any email.
      - If you check "Email empty logs" then if "Email sending" is selected, you will receive emails even if there is no commit.

  • Output message type

    • There is 2 kinds of logs presentations

      - "Filtered logs" means the logs are filtered according to your profile and the logs are presented per directory.
      - "Raw logs" means you get the svn or cvs 'log' output (or similar), there is no special processing on the output.

  • Commits log message format

    • And you can have HTML email or only TEXT email.

      - If you choose the HTML, you will get a HTML version of the log, with clickable link to get the diff, annotate/blame, revision information and so on.
      - Now if you choose the TEXT format, you will get either the Raw logs or a Text version of the Filtered logs presentation. There is no clickable link in the logs, but the email size will be much smaller (and newsgroups compliant).
